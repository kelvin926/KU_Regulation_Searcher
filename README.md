## 이 프로그램은 무엇인가요?

KU Regulation Searcher는 고려대학교 규정과 학칙을 내 컴퓨터에 저장해 두고 빠르게 검색할 수 있게 해주는 프로그램입니다.

예를 들어 이런 질문을 할 수 있습니다.

- 휴학은 몇 학기까지 가능한가요?
- 수료연구등록은 어떤 경우에 하나요?
- 제76조의2와 관련된 내용을 찾아줘.
- 학위청구논문 심사 조건을 알려줘.
- 세종 총학생회와 서울 총학생회의 차이를 알려줘.
- 미래모빌리티학과 학생이 군휴학을 냈다가 일반휴학으로 전환하고 싶대. 어떻게 해야 하지?
- 신임 교수의 영강 의무는 어떻게 되지?

앱은 먼저 내 컴퓨터에 저장된 규정에서 관련 조항을 찾고, 그 조항만 Gemini AI에 보내 답변을 만듭니다. 관련 근거가 없으면 억지로 답하지 않고 `[근거 없음]`으로 처리합니다.

0.9.1부터는 `규정 질의`에서 캠퍼스와 질의 그룹을 따로 고를 수 있습니다. 둘 다 기본값은 `자동 판단`입니다. 캠퍼스는 `서울캠퍼스`, `세종캠퍼스`, `기타` 중 하나로 좁힐 수 있고, 질의 그룹은 `학부생`, `일반대학원`, `전문·특수대학원`, `교원/교수`, `직원/조교`, `기타` 중 하나로 좁힐 수 있습니다. 예를 들어 `세종캠퍼스 / 학부생`, `서울캠퍼스 / 교원/교수`처럼 조합해서 검색할 수 있습니다.

학과 내규처럼 공식 규정 시스템에 없는 자료도 `저장 데이터` 화면에서 커스텀 규정으로 붙여넣어 등록할 수 있습니다. 커스텀 규정도 적용 캠퍼스와 적용 그룹을 따로 지정할 수 있습니다. 등록한 커스텀 규정은 공식 규정과 함께 검색되고, 근거 카드에는 `공식 규정` 또는 `커스텀 규정` 배지가 표시됩니다.

중요: 이 앱의 답변은 규정을 쉽게 찾아보기 위한 참고용입니다. 실제 행정 처리나 최종 판단은 학과사무실, 행정부서, 담당 부서에 확인하세요.

## 라이선스

이 프로그램의 소스 코드는 공개되어 있으며, 자유롭게 수정해서 사용할 수 있습니다.

다만 상업적 이용은 허용하지 않습니다.

고려대학교 규정, 학칙, 조문, 별표, 별첨, HWP/PDF 원문 등의 권리는 고려대학교 또는 해당 권리자에게 있습니다.

이 저장소는 규정 원문을 포함하지 않으며, 사용자가 접근 권한을 가진 규정을 개인 PC에서 검색하기 위한 도구입니다.

## 다운로드

1. [GitHub Releases](https://github.com/kelvin926/KU_Regulation_Searcher/releases)에 들어갑니다.
2. 최신 버전의 `KU-Regulation-Setup-0.9.1.exe`를 다운로드합니다.
3. 다운로드한 설치 파일을 실행합니다.
4. 설치가 끝나면 `KU Regulation Searcher`를 실행합니다.

별도로 Node.js, Python, 데이터베이스 프로그램을 설치할 필요가 없습니다.

Windows에서 SmartScreen 경고가 보일 수 있습니다. 현재는 코드 서명을 하지 않은 개인 배포 프로그램이라 생길 수 있는 경고입니다.

## 처음 사용하는 방법

처음에는 아래 순서대로 한 번만 준비하면 됩니다.

| 순서 | 할 일 | 설명 |
| --- | --- | --- |
| 1 | 로그인 | 고려대학교 공식 로그인 창에서 직접 로그인합니다. |
| 2 | AI 설정 | Google AI Studio에서 만든 Gemini API Key를 저장합니다. |
| 3 | 규정 목록 새로고침 | 앱이 동기화할 수 있는 규정 목록을 불러옵니다. |
| 4 | 규정 동기화 | 필요한 규정을 선택해서 내 컴퓨터에 저장합니다. |
| 5 | 검색 또는 질문 | 저장된 규정을 검색하거나 AI 답변을 생성합니다. |

### 1. 고려대 로그인

1. 앱 왼쪽 메뉴에서 `로그인`을 누릅니다.
2. `로그인 열기`를 누릅니다.
3. 새로 열린 고려대학교 공식 로그인 창에서 직접 로그인합니다.
4. 로그인이 끝나면 로그인 창을 닫습니다.
5. 앱의 로그인 상태가 `로그인됨`으로 바뀌었는지 확인합니다.

앱은 고려대 아이디와 비밀번호를 저장하지 않습니다.

### 2. 규정 목록 새로고침

1. 왼쪽 메뉴에서 `규정 동기화`를 누릅니다.
2. `규정 목록 새로고침`을 누릅니다.
3. 완료 안내가 뜨면 규정 목록을 폴더처럼 펼쳐 볼 수 있습니다.

규정 목록을 아직 새로고침하지 않았다면 먼저 새로고침하라는 안내가 표시됩니다.

### 3. 필요한 규정 동기화

1. 동기화할 규정을 체크합니다.
2. `선택 규정 동기화`를 누릅니다.
3. 진행률, 성공 수, 실패 수, 예상 소요 시간을 확인합니다.

규정을 전체 선택해서 동기화하면 약 1시간이 걸릴 수 있습니다. 네트워크 상태, 규정관리시스템 응답 속도, 선택한 규정 수에 따라 더 짧거나 길어질 수 있습니다.

일부 규정은 규정관리시스템에서 웹 본문 대신 HWP/PDF 원문 파일로만 제공됩니다. 이런 규정은 동기화 실패로 처리하지 않고, 검색 결과에 `원문 파일` 안내가 표시되며 HWP/PDF 다운로드 버튼으로 원문을 확인할 수 있습니다.

## Gemini API Key 준비하기

AI 답변을 사용하려면 Gemini API Key가 필요합니다. API Key는 Gemini AI를 내 계정으로 사용할 수 있게 해주는 개인용 키입니다.

### API Key 만들기

1. [Google AI Studio API Keys](https://aistudio.google.com/app/apikey)를 엽니다.
2. Google 계정으로 로그인합니다.
3. 약관 동의 화면이 나오면 내용을 확인하고 동의합니다.
4. `Create API key` 또는 `API 키 만들기`를 누릅니다.
5. 키가 만들어지면 `Copy` 버튼으로 복사합니다.

### 앱에 입력하기

1. 앱 왼쪽 메뉴에서 `AI 설정`을 누릅니다.
2. `Gemini API Key` 입력칸에 복사한 키를 붙여넣습니다.
3. 사용할 모델을 선택합니다.
4. `API Key 저장`을 누릅니다.
5. `연결 테스트`를 눌러 정상 동작하는지 확인합니다.

API Key가 저장되면 입력칸은 잠깁니다. 새 키로 바꾸고 싶으면 먼저 `API Key 삭제`를 누른 뒤 다시 입력하세요.

API Key 전체 값은 다른 사람에게 보내지 마세요. GitHub, 블로그, 카카오톡, 이메일, `.env` 파일에도 올리지 않는 것이 좋습니다.

### 어떤 모델을 고르면 되나요?

| 모델 | 이런 때 추천 | 알아둘 점 |
| --- | --- | --- |
| Gemini 3.1 Flash Lite | 대부분의 짧은 규정 질문, 연결 테스트, 빠른 확인 | Preview 모델이라 제공 여부와 사용 제한이 바뀔 수 있습니다. |
| Gemma 4 31B | Gemini 답변과 비교, 사용량 분산, 긴 답변 실험 | 근거 조항만 사용하고 형식을 지켜야 하는 답변에서는 편차가 있을 수 있습니다. |

`AI 설정` 화면에서는 규정 질의에서 보여줄 검색 후보 수와 AI 답변에 넘길 최대 근거 조항 수도 조정할 수 있습니다. 후보 수를 늘리면 근거를 더 넓게 볼 수 있지만, 답변 생성이 느려지고 토큰 사용량이 늘어날 수 있습니다.

기본값은 화면 검색 후보 30개, AI 최대 근거 조항 12개입니다. 앱은 넓게 찾은 뒤 질문 의도와 소속 범위를 기준으로 다시 정렬하고, AI에는 더 적고 정확한 근거만 보내도록 설계되어 있습니다.

## 검색하는 방법

`규정 검색` 또는 `규정 질의` 화면에서 검색어를 입력하고 Enter를 누르면 바로 검색됩니다.

| 이렇게 입력하면 | 이렇게 찾습니다 |
| --- | --- |
| `휴학` | 휴학이 들어간 조항 |
| `제76조의2` | 특정 조문 번호 |
| `"일반휴학"` | 정확히 같은 문구 |
| `휴학 OR 복학` | 휴학 또는 복학이 들어간 조항 |
| `휴학 -군입대` | 휴학은 포함하고 군입대는 제외 |

검색 결과에서 검색어와 일치하는 부분은 형광펜처럼 표시됩니다.

`방법`, `알려줘`, `규정`처럼 질문 의도를 나타내는 말은 검색어가 아니라 질문 유형으로 처리됩니다. 예를 들어 `복학하는 방법을 알려줘`는 `복학`, `복학신청`, `제출` 같은 조항을 우선 찾고, `포상의 방법`처럼 단어만 비슷한 조항은 낮게 봅니다.

학부와 대학원은 적용되는 규정이 다를 수 있습니다. `학부생 군입대`, `일반대학원 휴학`, `교육대학원 장학금`처럼 소속이나 과정을 함께 적으면 더 정확하게 찾습니다.

0.9.x에서는 아래처럼 여러 조건이 섞인 질문도 더 잘 처리하도록 검색 파이프라인을 보강했습니다.

| 질문 예시 | 앱이 함께 고려하는 근거 방향 |
| --- | --- |
| `세종 총학생회와 서울 총학생회의 차이를 알려줘` | 총학생회칙, 학칙의 학생자치 조항, 세종/서울 관련 사무분장 |
| `미래모빌리티학과 학생이 군휴학을 냈다가 일반휴학으로 전환하고 싶대` | 군입대 휴학, 특별휴학 기간, 일반휴학 기간과 신청 조항 |
| `신임 교수의 영강 의무는 어떻게 되지?` | 신임교원, 책임수업시간, 영어강의/외국어강의 관련 조항 |

`규정 질의` 화면에서 `캠퍼스`와 `질의 그룹`을 직접 고르면 자동 판단보다 선택한 범위를 더 강하게 우선합니다. 캠퍼스 선택은 `서울캠퍼스`, `세종캠퍼스`, `기타`로 따로 분리되어 있으며, 캠퍼스 규정과 공통 규정이 함께 필요할 때는 공통 규정도 후보에서 배제하지 않습니다.

## AI 답변을 받는 방법

1. `규정 질의` 화면에 질문을 입력합니다.
2. Enter를 누르거나 `관련 조항 찾기`를 누릅니다.
3. 앱이 관련 조항 후보를 보여줍니다.
4. 필요한 근거 조항을 확인합니다.
5. `AI 답변 생성`을 누릅니다.
6. 답변과 함께 AI가 사용한 조항이 표시됩니다.

AI 답변 아래에는 언급된 조항과 내용이 강조되어 표시됩니다. 근거가 부족하거나 선택한 조항 밖의 내용을 말하면 경고가 표시될 수 있습니다.

질문 범위가 넓으면 관련 조항이 화면에 보이는 후보보다 더 많을 수 있습니다. 예를 들어 휴학 기간은 학부, 대학원, 전문대학원, 학과 내규에 따라 다를 수 있으므로 앱이 `상위 후보 기준` 안내를 보이면 소속이나 대학원 이름을 함께 넣어 다시 검색하는 것이 좋습니다.

예: `일반대학원 복학`, `일반대학원 장학금`, `학사운영 규정 복학`

### 커스텀 규정 등록

공식 규정 시스템에 없는 학과 내규, 부서 안내, 학과별 운영 기준은 `저장 데이터` 화면의 `커스텀 규정` 섹션에서 등록할 수 있습니다.

1. `규정명`을 입력합니다.
2. `적용 캠퍼스`와 `적용 그룹`을 고릅니다.
3. 필요하면 `출처/메모`에 학과 공지명이나 적용 학기를 적습니다.
4. 본문을 붙여넣고 `커스텀 규정 등록`을 누릅니다.

본문에 `제1조`, `제2조의2` 같은 조문 제목이 있으면 자동으로 조항 단위로 나누어 저장합니다. 조문 패턴이 없으면 `전체`라는 단일 조항으로 저장합니다.

커스텀 규정은 기본적으로 `규정 질의` 검색과 AI 답변 근거에 포함됩니다. 필요하지 않은 질문에서는 `커스텀 규정 포함` 토글을 끄면 공식 규정만 검색합니다.

## 자주 묻는 질문

### 고려대 아이디와 비밀번호를 앱이 저장하나요?

아니요. 앱은 아이디와 비밀번호를 저장하지 않습니다. 사용자는 고려대학교 공식 로그인 창에서 직접 로그인합니다.

### 로그인 창에서 로그인한 뒤 뭘 해야 하나요?

로그인이 끝나면 로그인 창을 닫으면 됩니다. 창을 닫은 뒤 앱에서 로그인 상태를 확인합니다.

### 전체 동기화가 왜 오래 걸리나요?

규정 수가 많고, 고려대학교 규정관리시스템에 부담을 주지 않도록 한 번에 하나씩 천천히 가져오기 때문입니다. 전체 선택 시 약 1시간이 걸릴 수 있습니다.

### 어떤 규정은 `원문 파일`이라고만 보여요.

일부 규정은 규정관리시스템에서 웹 본문을 제공하지 않고 HWP/PDF 파일로만 제공합니다. 앱은 이런 규정을 실패로 넘기지 않고 저장해 두며, 원문은 규정 검색 화면의 다운로드 버튼으로 확인할 수 있습니다.

### AI가 틀릴 수도 있나요?

네. AI 답변은 참고용입니다. 앱이 근거 조항을 함께 보여주지만, 최종 판단은 담당 부서에 확인해야 합니다.

### 설치 파일 실행 시 Windows 경고가 떠요.

현재 배포 파일은 코드 서명이 없는 개인 배포 파일입니다. 그래서 SmartScreen 경고가 뜰 수 있습니다.

## 지원 모델

- Gemini 3.1 Flash Lite: `gemini-3.1-flash-lite-preview` - 빠른 응답과 형식 준수가 중요한 기본 선택입니다.
- Gemma 4 31B: `gemma-4-31b-it` - 답변 비교, 사용량 분산, 긴 답변 실험용 대체 선택입니다.

## AI 답변 방식

1. 사용자의 질문을 규정집 표현에 맞게 정규화하고, 복합 질문은 여러 검색 의도로 나눕니다.
2. 로컬 SQLite FTS5 검색, 규정명 검색, 조문명 검색, 보조 LIKE 검색을 함께 사용해 후보를 넓게 수집합니다.
3. 질문 의도, 소속 범위, 규정명/조문명 일치도, 핵심 조항 여부를 기준으로 후보를 다시 정렬합니다.
4. 관련도가 높은 조항 후보만 Gemini API로 보냅니다.
5. 전체 규정집을 모델에 보내지 않습니다.
6. 모델 응답의 `used_article_ids`를 로컬 DB 기준으로 다시 검증합니다.
7. 근거 후보에 없는 조항을 언급하면 경고를 표시합니다.

0.9.1 검색 파이프라인은 로컬 규정 DB 기반 자연어 복합 질문 10,000개 평가에서 10,000개 모두 기대 근거를 찾는 것으로 검증했습니다. 캠퍼스/질의 그룹 분리, 커스텀 규정 CRUD/검색, 설치 표시 버전 회귀 테스트도 함께 통과했습니다.

## 보안 원칙

- 고려대 아이디와 비밀번호는 저장하지 않습니다.
- 사용자는 고려대학교 공식 로그인 창에서 직접 로그인합니다.
- 로그인 우회, 권한 우회, 비인가 접근 기능은 없습니다.
- 사용자가 본인 계정으로 접근 가능한 규정만 수집합니다.
- Gemini API Key는 Electron safeStorage로 암호화해서 저장합니다.
- API Key, 쿠키, 세션, DB, 로그, `.env` 파일은 GitHub에 커밋하지 않습니다.

## 개발자용

Windows가 기준 개발 환경입니다.

```bash
npm install
npm test
npm run build
npm run rebuild:electron
npm run dist:win
git diff --check
```

개발 모드 실행:

```bash
npm run dev
```

Vite 개발 포트는 `6127`입니다. 이 Windows PC에서 기본 포트 `5173`이 excluded port range에 걸려 `EACCES`가 발생했기 때문에 변경했습니다.

Windows 검증 기록은 [TESTING.md](TESTING.md)에 정리되어 있습니다.
