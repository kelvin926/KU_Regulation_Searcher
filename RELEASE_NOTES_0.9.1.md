# KU Regulation Searcher 0.9.1

## 새 기능

- `규정 질의` 화면에 질의 그룹 선택 UI를 추가했습니다.
- 기본값은 `자동 판단`이며, 필요하면 `학부생`, `일반대학원`, `전문·특수대학원`, `교원/교수`, `직원/조교`, `서울캠퍼스`, `세종캠퍼스`, `기타`를 직접 선택할 수 있습니다.
- 캠퍼스 그룹을 `서울캠퍼스`, `세종캠퍼스`, `기타`로 분리했습니다. 캠퍼스 규정과 공통 규정이 함께 필요할 때는 공통 규정도 후보에서 배제하지 않습니다.
- `저장 데이터` 화면에 커스텀 규정 관리 섹션을 추가했습니다.
- 학과 내규나 부서 안내를 `규정명`, `적용 그룹`, `출처/메모`, `본문`으로 붙여넣어 등록할 수 있습니다.
- 커스텀 규정 본문은 `제N조`, `제N조의M` 패턴으로 자동 분리하고, 조문 패턴이 없으면 `전체` 단일 조항으로 저장합니다.
- 질의 화면에서 `커스텀 규정 포함` 토글을 제공하며 기본값은 켜져 있습니다.

## 개선 사항

- 선택한 질의 그룹을 검색 랭킹과 AI 답변 생성 요청에 함께 전달하도록 검색/AI 요청 인터페이스를 확장했습니다.
- 커스텀 규정도 공식 규정과 같은 검색 인덱스에 참여하도록 DB 구조를 확장했습니다.
- 커스텀 규정 수정·삭제 시 관련 조항과 FTS 인덱스를 함께 갱신합니다.
- 근거 카드와 AI가 언급한 조항 목록에 `공식 규정` / `커스텀 규정` 배지를 표시합니다.
- 공식 규정과 커스텀 규정이 함께 AI 근거로 전달될 때, 커스텀 규정은 사용자가 입력한 자료임을 프롬프트에 명시합니다.
- 서울/세종 총학생회 비교처럼 두 캠퍼스를 함께 묻는 질문에서는 한쪽 캠퍼스 선택 때문에 다른 캠퍼스 근거가 배제되지 않도록 보정했습니다.
